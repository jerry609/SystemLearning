# SystemLearning 项目结构指南

这是一个系统性学习计算机科学各个领域的综合性学习项目。项目采用渐进式学习路径，每个领域都有完整的实验室(lab)结构。

## 项目整体结构

主项目入口是 [README.md](mdc:README.md)，其中包含了项目的完整概述、学习理念和进度跟踪。

### 已完成的学习模块

#### Kubernetes 云原生容器编排
- **主文档**: [kubernetes/README.md](mdc:kubernetes/README.md)
- **学习总结**: [kubernetes/项目完成总结.md](mdc:kubernetes/项目完成总结.md)
- **实验体系**: 包含10个循序渐进的实验室，从基础概念到生产环境最佳实践

#### Docker 容器化技术
- **主文档**: [docker/README.md](mdc:docker/README.md)
- **实验体系**: 包含10个实验室，涵盖容器化技术的各个方面

## 学习路径组织原则

1. **渐进性**: 每个模块的实验室按照从基础到高级的顺序排列
2. **实践性**: 理论学习与动手实践相结合
3. **系统性**: 每个领域都有完整的知识体系覆盖

## 导航建议

- 当用户询问特定技术时，首先查看对应模块的主README文件
- 每个实验室都有独立的README.md，包含具体的学习内容和实践指导
- 使用项目总结文件了解学习成果和最佳实践

## 代码和配置文件处理

- 优先使用中文进行说明和注释
- 遵循每个技术栈的最佳实践
- 注重生产环境的安全性和可维护性
